# IEU-CEPA Compliance Scanner — Backend

FastAPI backend with a RAG (Retrieval-Augmented Generation) system that answers EU compliance questions for Indonesian SME exporters using indexed regulation PDFs.

## Setup

### 1. Install dependencies

```bash
cd backend
pip install -r requirements.txt
```

### 2. Configure environment

```bash
cp .env.example .env
# Open .env and replace "your-key-here" with your actual OpenAI API key
```

### 3. Add regulation documents

Place PDF files (EUDR, CE marking guides, DPP specs, ESG frameworks, etc.) into the `documents/` folder.

### 4. Index the documents

```bash
python -m rag.ingest
```

This reads all PDFs from `documents/`, splits them into chunks, creates OpenAI embeddings, and persists them to `chroma_db/`. Re-run whenever you add new PDFs.

### 5. Start the API server

```bash
uvicorn main:app --reload
```

The API is now available at `http://localhost:8000`.
Interactive docs: `http://localhost:8000/docs`

## Endpoints

| Method | Path | Description |
|--------|------|-------------|
| `GET` | `/health` | Health check |
| `POST` | `/chat` | RAG-powered compliance Q&A |
| `POST` | `/analyze-readiness` | Compliance readiness scan with score and action plan |

### POST /chat

```json
{
  "message": "What are the EUDR requirements for furniture exporters?",
  "history": []
}
```

Response:
```json
{
  "answer": "...",
  "sources": ["EUDR_Regulation_2023.pdf"]
}
```

### POST /analyze-readiness

```json
{
  "company_name": "PT Maju Jaya",
  "sector": "Furniture",
  "product_type": "Wooden chairs",
  "eudr_traceability": true,
  "eudr_deforestation_free": false,
  "eudr_due_diligence": false,
  "ce_marking": false,
  "ce_technical_file": false,
  "dpp_ready": false,
  "esg_reporting": false,
  "esg_carbon_footprint": false,
  "reach_compliance": true,
  "social_compliance": true
}
```

Response:
```json
{
  "readiness_score": 30,
  "risk_level": "High",
  "missing_requirements": ["EUDR Deforestation-free sourcing certified", "..."],
  "action_plan": ["Obtain EUDR due diligence certification", "..."],
  "analysis": "PT Maju Jaya shows early compliance steps but faces significant gaps..."
}
```

## Project structure

```
backend/
├── main.py              # FastAPI app and endpoint definitions
├── requirements.txt     # Python dependencies
├── .env.example         # Environment variable template
├── .env                 # Your actual secrets (not committed)
├── rag/
│   ├── __init__.py
│   ├── ingest.py        # PDF ingestion and ChromaDB indexing
│   └── retriever.py     # Vector retrieval and LLM response generation
├── documents/           # Place PDF regulation files here
│   └── .gitkeep
└── chroma_db/           # Auto-generated by ingest.py (not committed)
```

## Notes

- `chroma_db/` is generated automatically by `python -m rag.ingest` and should be added to `.gitignore`.
- The chat endpoint answers in the language the question is asked (German, English, or Indonesian).
- CORS is open to all origins — restrict `allow_origins` in `main.py` before deploying to production.
